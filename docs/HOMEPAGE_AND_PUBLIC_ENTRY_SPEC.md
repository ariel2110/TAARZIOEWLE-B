
# HOMEPAGE AND PUBLIC ENTRY SPEC

## Purpose
Define the public homepage where visitors, customers, and the admin can enter the system.

## Main goals
- Explain that the system automatically collects business information, detects gaps, and prepares demo websites.
- Provide clear login entry points for customers and admin.
- Provide an intake preview form where a business can enter available information and see what is missing for a strong demo website.
- Explain that customer demo creation is login-gated and limited to 2 demo websites per month.

## Homepage sections
1. Hero explaining the platform
2. How the automation works
3. What data the system collects and checks
4. Public intake preview form
5. Missing-data / readiness feedback
6. Customer login entry
7. Admin login entry
8. Privacy note

## Intake preview behavior
The user enters business information such as:
- business name
- city
- category
- phone
- address
- facebook
- instagram

The system returns:
- what was recognized
- what is missing
- readiness score
- suggested improvements
- whether the monthly demo limit is available
