
# PUBLIC COMPARE VIEWS SPEC

## Purpose
Provide a clearer comparison view between:
- current package/demo allowance
- onboarding session state
- recent demo request history

## V20 compare view
The compare view combines:
- availability
- latest summary
- recent demo request items

This helps both:
- the public flow
- the admin public-flow console

## Why it matters
Without a compare view, the user/operator sees fragmented state.
With a compare view, it becomes easier to understand:
- what the package currently allows
- what happened recently
- what the next action should be
